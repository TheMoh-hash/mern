# SneakerFlow - E-commerce FullStack (MERN)

Projet de fin d'apprentissage basé sur le cours "React, NodeJS, Express & MongoDB - The MERN Fullstack Guide" de Maximilian Schwarzmüller. Ce dépôt documente le développement d'une application e-commerce complète, mettant en œuvre une architecture FullStack robuste.

## 📝 Description du projet
SneakerFlow est une plateforme de vente de sneakers permettant aux utilisateurs de parcourir un catalogue, de gérer leur panier et de finaliser leurs commandes. Ce projet illustre ma capacité à concevoir et déployer une application web complète, de la modélisation de base de données à l'interface utilisateur interactive.

## 🚀 Fonctionnalités principales
- **Gestion des utilisateurs :** Authentification sécurisée par tokens (JWT) et hachage des mots de passe.
- **Catalogue & Recherche :** Navigation dynamique au sein du catalogue produit.
- **Panier d'achat :** Gestion de l'état global et persistance des données.
- **Interface Admin :** Espace dédié pour la gestion du CRUD (Create, Read, Update, Delete) des produits.

## 🛠 Stack Technique
- **Frontend :** React.js, Redux, Bootstrap CSS.
- **Backend :** Node.js, Express.js.
- **Base de données :** MongoDB, Mongoose (ODM).
- **Outils :** Axios (appels API), Multer (gestion d'images), JSON Web Token (sécurité).

## 🎓 Démarche académique
Dans le cadre de ce projet, j'ai particulièrement travaillé sur :
- **L'architecture MVC :** Séparation claire entre la logique métier, les routes et les vues.
- **La robustesse du code :** Mise en place d'une gestion d'erreurs cohérente côté serveur.
- **La sécurité :** Protection des routes API et validation des données entrantes.

## ⚙️ Installation
Pour installer et exécuter ce projet localement :

1. Cloner le dépôt : `git clone https://github.com/TheMoh-hash/mern.git`
2. Installer les dépendances : `npm install` (dans les dossiers `/backend` et `/frontend`)
3. Configurer le fichier `.env` avec vos variables d'environnement (`MONGO_URI`, `JWT_SECRET`).
4. Lancer le backend : `npm run dev`
5. Lancer le frontend : `npm start`
